Everything your OS needs. AI-native.

AIOS-Lite is not a chatbot wrapper — it is a complete operating system layer with a scheduler, permissions model, service registry, and a cognitive AI agent.

🤖

AI-Powered Shell (AURA)

Natural language OS commands via local LLaMA LLM or rule-based fallback — no cloud, no subscription, no telemetry.

📱

Cross-OS Bridge

Mirror iOS, Android, Linux, macOS, and remote SSH hosts under a single unified filesystem — all from your terminal.

🧠

Hybrid Memory

Three-layer memory: context window, symbolic key-value store, and semantic embeddings. Your AI remembers everything you tell it.

🌐

HTTP REST / WebSocket API

Built-in os-httpd server with full REST, SSE, and WebSocket endpoints. Control your OS from any browser or script.

🔒

Secure by Design

Capability-based permissions, OS_ROOT filesystem jail, and HMAC audit logs. Every action is controlled and traceable.

📦

Truly Portable

Runs from a USB drive, a $15 Raspberry Pi Zero, an Android phone, a cloud VPS, or bare metal x86 hardware.

🔧

Self-Healing System

Five-stage recovery mode, health monitoring, automatic log rotation, and os-recover auto-repair — the OS fixes itself.

📋

Service Registry

PID-1 boot init, cooperative round-robin scheduler, service health checks, and event bus — a real kernel, in POSIX shell.

🔌

Plugin System

Drop shell scripts into OS/lib/aura-mods/ to extend the OS at runtime. Hot-reload coming in v1.0.

Up and running in 60 seconds

No build step required. Works on any POSIX system with Python 3.8+ and Bash.

$ git clone https://github.com/Cbetts1/PROJECT.git && cd PROJECT
$ bash install.sh
$ sh OS/sbin/init
[ AIOS-Lite ] Booting kernel… OK
[ AIOS-Lite ] Starting AURA agent… OK
[ AIOS-Lite ] Services: 7/7 healthy
$ os-shell
aios> ask what is my system status
AURA: All 7 services are healthy. CPU 4%, memory 18%, disk 22%. LLM backend: llama.cpp 7B int4 — ready.
aios> bridge.detect
AURA: Detected 1 Android device (USB). Run mirror.mount android to browse it.
