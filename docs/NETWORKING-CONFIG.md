## 1. Network Configuration Templates

### 1.1 Network Config Snapshot (`OS/etc/net.conf`)

AIOS-Lite stores a persistent network snapshot via `os-netconf save`. The snapshot file is a shell-sourceable key-value format:

```sh
# OS/etc/net.conf — generated by: os-netconf save
AIOS_NET_SNAPSHOT_TIME="2026-04-03T15:55:10Z"

# Primary interface
AIOS_IFACE="eth0"
AIOS_IP="192.168.1.50/24"
AIOS_GW="192.168.1.1"
AIOS_DNS="1.1.1.1"

# WiFi
AIOS_WIFI_IFACE="wlan0"
AIOS_WIFI_SSID="MyNetwork"
AIOS_WIFI_CONNECTED="true"

# Bluetooth
AIOS_BT_ENABLED="true"
AIOS_BT_DEVICE="hci0"

# NAT
AIOS_NAT_ENABLED="false"
AIOS_NAT_WAN_IFACE="eth0"

# Firewall
AIOS_FIREWALL_ENABLED="true"
```

**Save and load:**

```sh
# Save current configuration
os-netconf save /opt/aios/OS/etc/net.conf

# Restore at boot (add to OS/etc/init.d/os-kernel or rc2.d)
. /opt/aios/OS/etc/net.conf
os-netconf ip set "$AIOS_WIFI_IFACE" "$AIOS_IP"
os-netconf route add default "$AIOS_GW"
os-netconf dns set "$AIOS_DNS"
```

## 2. WiFi Integration Points

AIOS-Lite manages WiFi through `OS/bin/os-netconf wifi`, which wraps `nmcli` (NetworkManager) on Linux or `iw`/`wpa_supplicant` on minimal systems.

### 2.1 WiFi Operations

```sh
# View current WiFi state
os-netconf wifi status

# Scan for nearby access points
os-netconf wifi scan

# Connect to a network
os-netconf wifi connect "MyNetwork" "my-password"

# Connect to an open network
os-netconf wifi connect "CoffeeShop_WiFi"

# Disconnect
os-netconf wifi disconnect
```

### 2.2 nmcli Wrapper (used internally)

```sh
# Underlying calls used by os-netconf wifi
nmcli device wifi list
nmcli device wifi connect "SSID" password "pass"
nmcli device disconnect wlan0
nmcli -t -f ACTIVE,SSID dev wifi | grep '^yes'
```

### 2.3 wpa_supplicant Template (minimal / embedded)

```ini
# /etc/wpa_supplicant/wpa_supplicant.conf
ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1
country=US

network={
    ssid="MyNetwork"
    psk="my-password"
    key_mgmt=WPA-PSK
    priority=10
}

network={
    ssid="BackupNetwork"
    psk="backup-password"
    key_mgmt=WPA-PSK
    priority=5
}
```

### 2.4 WiFi Integration for Cross-OS Bridge

When bridging to a remote device over SSH/SSHFS, AIOS-Lite requires an active WiFi or Ethernet connection:

```sh
# Ensure WiFi is up before mounting remote bridge
os-netconf wifi connect "HomeNet" "password"
os-mirror mount ssh user@192.168.1.100
```

## 3. Bluetooth Integration Points

AIOS-Lite manages Bluetooth through `OS/bin/os-netconf bt`, which wraps `bluetoothctl`.

### 3.1 Bluetooth Operations

```sh
# Check Bluetooth adapter status
os-netconf bt status

# Scan for nearby BT devices
os-netconf bt scan
```

### 3.2 bluetoothctl Wrapper (used internally)

```sh
# Underlying calls used by os-netconf bt
bluetoothctl show          # Adapter info
bluetoothctl scan on       # Start discovery
bluetoothctl devices       # List discovered devices
bluetoothctl pair  <MAC>   # Pair with a device
bluetoothctl connect <MAC> # Connect to paired device
```

### 3.3 Bluetooth PAN (Personal Area Network)

To use a Bluetooth device as a network interface:

```sh
# Create BT PAN interface (requires bluez + bnep)
modprobe bnep
bt-network connect <MAC> nap
os-netconf ip set bnep0 10.0.0.1/24
```

### 3.4 Bluetooth for iOS Bridge

When using the iOS bridge (`os-bridge ios`), Bluetooth pairing is managed by `libimobiledevice`:

```sh
os-bridge ios pair     # Pairs over USB first, then trusts BT
os-bridge ios detect   # Detects paired iOS devices over BT or USB
```

### 3.5 Bluetooth Integration Notes

| Platform | Tool | Notes |
|----------|------|-------|
| Linux / Raspberry Pi | `bluetoothctl`, `rfcomm` | Full support |
| Android (Termux) | Android BT API (not directly accessible) | Use ADB bridge instead |
| macOS | `blueutil` (via Homebrew) | Partial support |

## 4. IP Assignment Strategy

### 4.1 DHCP (Default)

On most deployments, the host OS provides DHCP-assigned addresses. AIOS-Lite reads them from `ip addr` / `ifconfig`.

### 4.2 Static IP Assignment

```sh
# Assign static IP via os-netconf
os-netconf ip set eth0 192.168.1.50/24
os-netconf route add default 192.168.1.1
os-netconf dns set 1.1.1.1
```

### 4.3 Recommended IP Schema (LAN)

```
192.168.1.1       — Router / default gateway
192.168.1.2       — AIOS-Lite API server (VPS or Raspberry Pi)
192.168.1.3–49    — DHCP pool (dynamic clients)
192.168.1.50      — AIOS-Lite static (if self-hosted)
192.168.1.51–100  — Reserved for future AIOS nodes
192.168.1.101–254 — General devices
```

### 4.4 IPv6

IPv6 is supported via the host OS. AIOS-Lite network commands (`os-netconf ip`, `os-netconf route`) accept CIDR notation for both IPv4 and IPv6:

```sh
os-netconf ip set eth0 2001:db8::50/64
os-netconf route add ::/0 2001:db8::1
```

### 4.5 Tailscale / WireGuard (recommended for remote access)

For secure remote access without exposing the server to the public internet:

```sh
# Install Tailscale
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

# AIOS-Lite binds to Tailscale IP automatically
os-netconf interfaces   # Shows tailscale0 with 100.x.x.x address
```

## 5. Routing Table Structure

### 5.1 View Routing Table

```sh
os-netconf route show
# Internally calls: ip route show
```

**Example output:**

```
default via 192.168.1.1 dev eth0
10.0.0.0/8 via 10.0.0.1 dev tun0
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.50
100.64.0.0/10 dev tailscale0 scope link
```

### 5.2 Route Management

```sh
# Add a static route
os-netconf route add 10.10.0.0/16 192.168.1.254

# Remove a route
os-netconf route del 10.10.0.0/16

# Add default gateway
os-netconf route add default 192.168.1.1
```

### 5.3 Policy Routing (advanced)

For multi-homed deployments (WiFi + Ethernet + Tailscale):

```sh
# Add routing table for WiFi traffic (table 100)
ip rule add from 192.168.1.50 table 100
ip route add default via 192.168.1.1 dev eth0 table 100
ip route add default via 192.168.2.1 dev wlan0 table 200
```

### 5.4 Routing Table for Cross-OS Bridge

When SSH/SSHFS bridge is active (`mirror/linux/`), AIOS-Lite communicates with remote hosts over the existing routes. No additional routing configuration is needed unless the remote is on a non-routable subnet.

```
Local host  ──(eth0: 192.168.1.50)──► Router (192.168.1.1)
                                            │
                                      ──► Remote SSH host (192.168.2.10)
                                            │
                                      (mirror/linux/ssh_192.168.2.10/)
```

## 6. Firewall Rule Recommendations

AIOS-Lite manages iptables firewall rules via `os-netconf firewall`.

### 6.1 Firewall Operations

```sh
# View firewall status
os-netconf firewall status

# Enable / disable
os-netconf firewall enable
os-netconf firewall disable

# List rules
os-netconf firewall rules

# Add a custom rule
os-netconf firewall add "INPUT -p tcp --dport 8080 -j ACCEPT"

# Flush all rules
os-netconf firewall flush
```

### 6.2 Recommended iptables Ruleset

The following policy is recommended for a VPS or Raspberry Pi running AIOS-Lite:

```sh
#!/usr/bin/env bash
# scripts/firewall-setup.sh — AIOS-Lite recommended firewall rules

# Default policies
iptables -P INPUT   DROP
iptables -P FORWARD DROP
iptables -P OUTPUT  ACCEPT

# Allow established/related connections
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Allow loopback
iptables -A INPUT -i lo -j ACCEPT

# Allow SSH (change port if using non-standard)
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Allow HTTP/HTTPS (public-facing reverse proxy)
iptables -A INPUT -p tcp --dport 80  -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# AIOS-Lite API server — localhost only (nginx proxies externally)
iptables -A INPUT -p tcp -s 127.0.0.1 --dport 8080 -j ACCEPT
iptables -A INPUT -p tcp -s 127.0.0.1 --dport 8443 -j ACCEPT

# ICMP (ping) — rate-limited
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 5/s -j ACCEPT

# Drop everything else
iptables -A INPUT -j DROP

# Save rules
iptables-save > /etc/iptables/rules.v4
```

### 6.3 UFW Shorthand (Ubuntu)

```sh
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw allow 80/tcp
ufw allow 443/tcp
# Do NOT expose 8080 publicly — nginx handles it
ufw enable
```

### 6.4 Firewall Rule Summary Table

| Direction | Protocol | Port | Source | Action | Purpose |
|-----------|----------|------|--------|--------|---------|
| INPUT | TCP | 22 | Any | ACCEPT | SSH administration |
| INPUT | TCP | 80 | Any | ACCEPT | HTTP (redirect to HTTPS) |
| INPUT | TCP | 443 | Any | ACCEPT | HTTPS (reverse proxy) |
| INPUT | TCP | 8080 | 127.0.0.1 | ACCEPT | API server (internal only) |
| INPUT | TCP | 8443 | 127.0.0.1 | ACCEPT | API server TLS (internal) |
| INPUT | ICMP | — | Any | ACCEPT (rate-limited) | Ping / diagnostics |
| INPUT | ALL | ALL | Any | DROP | Default deny |
| OUTPUT | ALL | ALL | Any | ACCEPT | All outbound allowed |
| FORWARD | ALL | ALL | Any | DROP | No routing (unless NAT enabled) |

## 7. NAT / Port Mapping Examples

### 7.1 Enable NAT (IP Masquerade)

AIOS-Lite supports NAT via `os-netconf nat`, which uses iptables masquerade:

```sh
# Enable NAT on the WAN interface (share internet over eth0)
os-netconf nat enable eth0

# Disable NAT
os-netconf nat disable

# View NAT status
os-netconf nat status
```

**Underlying iptables commands:**

```sh
# Enable IP forwarding
echo 1 > /proc/sys/net/ipv4/ip_forward

# Add masquerade rule
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
iptables -A FORWARD -i wlan0 -o eth0 -j ACCEPT
iptables -A FORWARD -i eth0 -o wlan0 -m state --state RELATED,ESTABLISHED -j ACCEPT
```

### 7.2 Port Forwarding Examples

Route external traffic to internal AIOS services:

```sh
# Forward external port 443 → AIOS API server at 127.0.0.1:8080
iptables -t nat -A PREROUTING -p tcp --dport 443 -j REDIRECT --to-port 8080

# Forward external port 8443 → AIOS HTTPS server at 127.0.0.1:8443
iptables -t nat -A PREROUTING -p tcp --dport 8443 -j DNAT --to-destination 127.0.0.1:8443

# Port forward 2222 (external) → 22 (internal SSH)
iptables -t nat -A PREROUTING -p tcp --dport 2222 -j DNAT --to-destination 192.168.1.50:22
```

> **Recommended approach:** Use a reverse proxy (nginx/Caddy) on port 443 instead of raw port forwarding. This gives you TLS termination, logging, and rate limiting.

### 7.3 NAT for Cross-OS Bridge (Phone as Gateway)

When AIOS-Lite runs on an Android phone (Termux) connected to WiFi, and a Raspberry Pi connects over USB tethering:

```
Raspberry Pi ──(USB0: 192.168.42.2)──► Android Phone (192.168.42.1)
                                              │ WiFi NAT
                                        ──► Internet
```

On the Android phone (Termux with root or via ADB):

```sh
# Enable USB tethering NAT
ip route add default via 192.168.1.1  # phone's WiFi gateway
iptables -t nat -A POSTROUTING -o wlan0 -j MASQUERADE
echo 1 > /proc/sys/net/ipv4/ip_forward
```

### 7.4 Port Mapping Table

| External Port | Internal Host | Internal Port | Protocol | Service |
|---------------|---------------|---------------|----------|---------|
| 443 | 127.0.0.1 | 8080 | TCP | AIOS API (via nginx/Caddy) |
| 8443 | 127.0.0.1 | 8443 | TCP | AIOS API TLS direct |
| 22 | 127.0.0.1 | 22 | TCP | SSH administration |
| 2222 | 192.168.1.50 | 22 | TCP | Remote SSH (port-forwarded) |
| 80 | 127.0.0.1 | 80 | TCP | HTTP → HTTPS redirect |

### 7.5 Persistent NAT / Firewall Rules

```sh

iptables-save > /etc/iptables/rules.v4
ip6tables-save > /etc/iptables/rules.v6

# Rules are auto-restored at boot via netfilter-persistent.service
```
